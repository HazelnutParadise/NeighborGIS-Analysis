# NeighborGIS 納博聚思 都市更新資訊整合平台 企劃書

# 一、創業理念及構想

## 1.1 創業動機與背景

都市更新是一座城市邁向現代化與永續發展不可或缺的一環。截至 2024 年，臺北市屋齡超過 30 年的老舊住宅已佔全市約四成，這些建築不僅普遍存在居住環境品質低落的問題，其能源效率低下的特性，更在全球氣候變遷的背景下，加劇了城市的碳排放壓力。考量建築部門佔全球近 40% 的碳排放量，推動都市更新並結合淨零建築的概念，不僅能改善居住品質，更是實現城市減碳目標的關鍵路徑。

## 1.2 市場痛點與問題癥結

儘管都市更新至關重要，現行的推動與估價流程卻存在諸多問題，大幅降低了民眾理解與參與的意願：

1.  **高成本與低效率**：傳統估價需委託專業人士，費用高昂且耗時甚久，使得一般民眾難以輕易獲取自身房產的價值評估。
2.  **資訊不透明且分散**：估價報告內容專業、權益分配規則複雜，加上更新後的建築樣貌、公設比、周邊環境特性等關鍵資訊不易取得或理解，容易造成資訊落差與信任危機。
3.  **永續效益認知不足**：民眾普遍缺乏對於綠建材、節能設計等淨零建築方案長期效益（如節省電費、提升生活品質、增加房屋價值）的具體認知，影響其採納意願。
4.  **缺乏整合溝通平台**：都市更新涉及多方利害關係人（居民、建商、政府、專業人士等），卻缺乏一個有效整合資訊、促進透明溝通與建立共識的數位平台。

## 1.3 解決方案：NeighborGIS 納博聚思

為解決上述痛點，「NeighborGIS 納博聚思」應運而生。我們致力於打造一個專為民眾設計的線上都市更新資訊整合與評估平台，核心目標是讓都市更新的資訊變得 **透明、易懂、易取得**。透過以下核心功能模組，我們希望能賦能居民，鼓勵更廣泛的社會參與：

*   **降低門檻，提升效率**：核心的「**厝邊好估道**」模組，運用 AI 與 GIS 技術分析公開數據，提供民眾快速、客觀且低成本的線上初步估價參考。
*   **資訊透明，易於理解**：「**我的鄰里DNA**」以視覺化方式呈現地點環境、人口及規劃資訊；「**厝邊好估道**」解析影響價格的關鍵因子；「**聚思空間**」提供建築配置參考，具體化更新後的空間想像。
*   **整合資訊，促進溝通**：平台整合**地點時事**、**政策法規**、**建商資訊**及**綠建材選項**等知識庫，並提供討論機制，促進多方利害關係人的對話與共識。

## 1.4 創新理念與發展願景

我們的核心創新在於技術整合與資訊簡化，利用 **AI、網路爬蟲與 GIS 技術**建立估價模型，並透過**視覺化圖表與房型建議引擎**，提供個人化的資訊服務。

「NeighborGIS 納博聚思」的願景是成為台灣都市更新領域最具公信力與影響力的資訊平台，促進各方理解與協作、降低估價爭議、推動淨零建築普及，最終為台灣的永續發展貢獻一份心力。

# 二、產品或服務內容

## 2.1 產品/服務概述

「NeighborGIS 納博聚思」是一個跨平台的線上系統，旨在整合都市更新相關的估價工具、環境分析與建築配置建議，協助民眾在複雜的決策過程中，能更有效率地獲取資訊，做出明智的選擇。

## 2.2 核心功能模組

平台主要由以下三大功能模組構成：

### 2.2.1 模組一：我的鄰里DNA（地點環境分析）

*   **功能**：使用者輸入地址後，平台自動進行地理編碼，並與都市計畫、基礎設施、潛在風險等多維度圖資進行空間分析，生成視覺化的地點分析報告。同時，透過 AI 搜尋引擎整合區域相關新聞，提供使用者對地點的全面性認識。
*   **效益**：將複雜的地理與環境資訊轉化為易懂的視覺化圖表，降低一般民眾的理解門檻，節省自行查詢彙整的時間，並提升開放資料的應用價值。

### 2.2.2 模組二：聚思空間（建物配置模擬）

*   **功能**：使用者輸入基地面積、預計戶數、公設比、陽台深度等參數後，系統能自動生成多種可能的建築平面配置參考圖（支援六種常見佈局模式）。
*   **效益**：提供具體的建築規劃參考，協助使用者了解更新後可能的樓層配置、空間利用與居住密度，將抽象的規劃藍圖具象化。

### 2.2.3 模組三：厝邊好估道（AI 輔助估價）

*   **功能**：採用特徵價格法（Hedonic Pricing Model），以 2022–2023 年台北市實價登錄資料為基礎，建構半對數迴歸模型。模型不僅考量傳統不動產屬性（如屋齡、樓層、建物型態），更創新地納入「我的鄰里DNA」分析出的基地內在特徵與周邊 500 公尺區位類型（透過 K-Means 分群）作為補充變數，以更精準地反映區位價值差異。
*   **效益**：提供使用者一個快速、客觀且低成本的更新前房地產價值初步評估參考，作為參與都市更新討論的基礎資訊。

## 2.3 技術架構

*   **前端（Frontend）**：採用 HTML, CSS, JavaScript 等標準網頁技術，確保跨裝置（桌機、行動裝置）的良好使用者體驗。
*   **後端（Backend）**：選用 Python 搭配 FastAPI 框架，處理 API 請求、業務邏輯與資料運算。
*   **資料庫（Database）**：使用 PostgreSQL 結合 PostGIS 擴充套件，高效處理地理空間資料的儲存與查詢。
*   **核心技術**：整合 Scikit-learn/TensorFlow (AI 模型)、BeautifulSoup/Scrapy (網頁爬蟲)、GeoPandas, Leaflet, OpenLayers, ECharts (GIS 分析與視覺化) 等工具。
*   **部署架構**：初期規劃使用自建伺服器，保留未來遷移至 AWS, Google Cloud, Azure 等公有雲平台的彈性。

## 2.4 商業模式與營運模式

我們採取**核心功能完全免費**的策略，不設立付費牆，以最大化資訊普及率與使用者基數。平台的營運收益主要來自以下幾個方面：

1.  **網頁廣告收益**：在平台頁面設置廣告版位，根據用戶輪廓與瀏覽內容，精準投放相關性高的廣告（如建商、房仲、家居、金融服務等）。
2.  **智慧推薦合作**：在環境分析、估價結果或建築配置頁面，根據用戶特性與需求，推薦合作夥伴的產品或服務（如建案、房仲物件、智慧家居、裝潢設計、金融方案等），透過導流或成交分潤獲利。
3.  **配置生成商品推薦**：「聚思空間」生成的配置圖旁，智能推薦適合該空間尺寸與風格的家具、家電、建材等商品，提供情境式購物體驗並獲取銷售分潤。
4.  **專業估價轉介**：對於有進一步精確估價需求的用戶，可轉介至合作的專業估價事務所，並收取轉介費用。
5.  **加值服務**：針對特定企業或政府單位需求，提供**客製化數據分析**或**專案合作**報價。

此模式旨在透過免費服務吸引大量用戶，再藉由廣告和多元合作夥伴關係實現營收，支持平台的永續發展。同時，堅持開放性，確保弱勢族群也能受益，推動產業透明化，善盡社會責任。

# 三、市場與競爭分析

## 3.1 市場特性與規模

台灣都市更新市場規模龐大且具備高度潛力，每年核准案件數與總投資金額均相當可觀。不同客群對平台的需求各有側重：個人用戶需要易用、低成本的工具；實施者需要更深入的分析功能；政府則可能需要政策模擬能力。

## 3.2 目標市場與客群

我們的目標市場鎖定在廣大的都市更新潛在參與者，特別是以下幾類關鍵客群：

| ------------------------ | ---------------------------------------------------- |
| **個人所有權人**         | 需要透明、易懂的估價資訊，了解權利變換規則，評估參與利弊 |
| **都市更新實施者** (建商) | 需要快速評估開發可行性，預估價值，有效與居民溝通       |
| **政府單位**             | 需要客觀工具評估政策成效，掌握民意，促進都更推動       |

    *   **核心定位**：「**都市更新的民眾資訊賦能平台**」

# 四、行銷策略

## 4.1 目標族群溝通策略

我們的首要目標是觸及廣大對都市更新議題相對陌生的都市居民。都更通常被視為專業門檻高的領域，複雜法規與術語讓民眾卻步。然而，都更結果會直接影響居民的生活品質與財產權益。「NeighborGIS 納博聚思」的核心使命便是**普及化都更知識**，降低參與門檻，讓民眾能輕鬆獲取資訊、評估利弊，進而積極參與，共同塑造更宜居的都市環境。

我們特別關注以下群體，並針對其特性進行溝通：

*   **老舊社區居民（非專業人士）**
    *   **溝通重點**：強調「家的延續」、「預見未來」，提供簡化資訊與成功案例分享，建立安全感。
    *   **需求**：對自家未來感到不確定，需要簡單工具評估參與都更的機會與風險。
*   **有意願但缺知識者**
    *   **溝通重點**：提供系統性學習資源，講解流程、規則與權益保障方法。
    *   **需求**：已意識到都更重要性，需要知識支援以做出明智決策。
*   **關心都市發展者**
    *   **溝通重點**：提供最新動態、政策解讀，鼓勵參與公共討論。
    *   **需求**：希望了解都更動態，參與公共討論，提升整體城市品質。
*   **建商與開發商**
    *   **溝通重點**：強調「效率提升」、「降低成本」、「客戶連結」的商業價值。
    *   **需求**：需要快速評估工具與接觸潛在客戶的橋樑。
*   **政府單位**
    *   **溝通重點**：強調「公共利益」、「政策支持」、「促進溝通」的社會價值。
    *   **需求**：需要評估政策、掌握民意、促進溝通的工具。

    *   **活動合作**：與政府單位、社區組織合作，在都更說明會、里民大會等場合介紹與推廣平台。
    *   **實體講座/工作坊**：自行或與專家合作舉辦都更主題講座，提升民眾認知。
    *   **夥伴推廣**：與房仲、建築師、估價師等專業人士合作，將平台推薦給其客戶。

## 4.4 通路策略：以網站為核心

**網站** 是我們提供服務與資訊的主要通路。我們將確保：

*   **使用者友善介面 (UI/UX)**：設計直觀、易於操作的介面。
*   **響應式網頁設計 (RWD)**：確保在桌機、平板、手機等不同裝置上均有良好瀏覽體驗。
*   **未來擴展**：視資源與合作情況，評估開發行動應用程式 (App) 的可行性，以擴大觸及範圍。

## 4.5 五感體驗行銷

透過刺激使用者的五種感官，加深平台的使用體驗與印象：

*   **視覺**：高品質動態圖表、精心設計的 UI。
*   **聽覺**：平台操作音效、教學影片旁白、Podcast 內容。
*   **觸覺**：線下活動的實體模型、流暢的觸控操作。
*   **嗅覺/味覺延伸**：線下活動結合環保建材氣味體驗、社區茶會等。

# 五、預期效益

## 5.1 質化效益

*   **提升估價效率與透明度**：AI 模型提供快速客觀參考，公開參數依據。
*   **減少估價爭議**：降低資訊不對稱，促進溝通與共識。
*   **促進民眾參與**：降低知識門檻，提供易用工具，提升參與意願。
*   **提升都更公平性**：客觀資訊有助於更公平的權利分配。
*   **降低學習成本**：簡化資訊呈現，普及都更知識。
*   **促進產業合作**：作為資訊整合平台，促進跨領域交流。

## 5.2 量化效益 (關鍵績效指標 KPI)

*   **平台流量與活躍度**：
    *   目標：首年 **月活躍用戶 (MAU, Monthly Active Users)** 達 5,000人，平均停留 8 分鐘；第三年 MAU 達 35,000人，月均訪問 3 次。
*   **使用者互動深度**：
    *   追蹤：估價工具使用次數、配置生成次數、社群討論活躍度。
*   **合作夥伴網絡**：
    *   目標：逐年穩定增加合作的建商、房仲、專業機構數量。
*   **都市更新影響力**：
    *   目標：協助至少 10 個案例溝通流程縮短 20%，爭議率降低 30%，居民同意率提升 15%。
*   **社會影響力**：
    *   目標：協助 5,000 位資源較少民眾獲取知識，知識落差降低 25%；平均節省估價相關費用 70%。
*   **財務表現**：
    *   目標：首年收支平衡，第二年營收成長 40%，利潤率 15%；第三年營收成長 35%，利潤率 20%。

## 5.3 效益評估方法

我們將採用多元方法持續評估平台效益：

1.  **使用者滿意度調查**：定期透過問卷、訪談收集回饋，目標第三年滿意度達 90% 以上。
2.  **平台使用數據分析**：利用網站分析工具追蹤流量、互動等 KPI。
3.  **都市更新案件追蹤**：與合作夥伴或透過公開資訊，追蹤平台介入案件的進展與成效。
4.  **財務報表分析**：定期檢視營收、成本、利潤等財務指標。
5.  **社會影響評估**：透過特定調查或數據分析，評估在資訊平權、成本節省等方面的社會貢獻。

## 5.4 預估損益

由於平台初期營收主要依賴廣告與合作推薦，具體金額較難精確預測。在編製詳細的預估損益表時，將採取**保守估計**原則，並根據實際營運狀況與市場反應，**滾動式修正**財務預測。初期重點將放在用戶增長與產品完善，並積極拓展合作夥伴關係以建立穩定的營收基礎。
